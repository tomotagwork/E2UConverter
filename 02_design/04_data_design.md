# データ設計書 — E2UConverter

## 1. コマンドラインオプション仕様

| オプション | 必須/任意 | 型 | デフォルト値 | 値の制約 |
|---|---|---|---|---|
| `-i <path>` | 必須 | String（パス） | — | 存在するファイルまたはディレクトリ |
| `-o <dir>` | 必須 | String（パス） | — | 書き込み可能なディレクトリパス（存在しない場合は自動作成） |
| `-c <codepage>` | 任意 | String | `IBM-930` | `Charset.forName()` で認識可能なコードページ名 |
| `-m <mode>` | 任意 | String | `fixed` | `fixed` または `stream` のいずれか |
| `-l <length>` | 任意 | int | `80` | 1 以上の正整数 |
| `-r` | 任意 | boolean フラグ | `false` | 引数なし（指定で `true`） |
| `-e <ext>` | 任意 | String | `null` | ドットなしの拡張子文字列（例: `txt`） |
| `-R <file>` | 任意 | String（パス） | `<outputDir>/report.md` | 書き込み可能なファイルパス |

---

## 2. 入力ファイル仕様

### 2-1. ファイル構造

| モード | レコード区切り | 1レコードのサイズ |
|---|---|---|
| 固定長（`fixed`） | なし（バイト数で分割） | `-l` オプション値（デフォルト 80 バイト） |
| バイトストリーム（`stream`） | EBCDIC CR/NL/NL25 | 可変長 |

### 2-2. EBCDIC 行区切りコード（バイトストリームモード）

| EBCDIC コード | 意味 | UTF-8 変換後 |
|---|---|---|
| `0x0D` | CR（Carriage Return） | `0x0A`（LF） |
| `0x15` | NL（New Line） | `0x0A`（LF） |
| `0x25` | NL25（New Line 2） | `0x0A`（LF） |

### 2-3. DBCS シフトコード

| EBCDIC コード | 意味 | 処理 |
|---|---|---|
| `0x0E` | SO（Shift-Out） | DBCS 開始。UTF-8 には出力しない。 |
| `0x0F` | SI（Shift-In） | DBCS 終了。UTF-8 には出力しない。 |

---

## 3. 不正文字検知対象コード一覧

### 3-1. 固定長レコードモード

| 範囲 | EBCDIC コード（16進） | 判定理由 | 備考 |
|---|---|---|---|
| `0x00`–`0x0D` | 00, 01, ..., 0D | 制御文字 | 0x0D（CR）も固定長では不正 |
| `0x10`–`0x3F`（一部除外） | 10, 11, ..., 3F | 制御文字・変換不能 | 0x0E（SO）、0x0F（SI）は除外 |
| `0xFF` | FF | 変換不能 | |

**除外コード（不正検知対象外）**

| コード | 理由 |
|---|---|
| `0x0E` | SO（DBCS 開始）: SO/SI 検証で別途評価 |
| `0x0F` | SI（DBCS 終了）: SO/SI 検証で別途評価 |
| `0x40`–`0xFE` | 通常の EBCDIC 文字コード範囲 |

### 3-2. バイトストリームモード

| 範囲 | EBCDIC コード（16進） | 判定理由 | 備考 |
|---|---|---|---|
| `0x00`–`0x0C` | 00, 01, ..., 0C | 制御文字 | |
| `0x10`–`0x14`（一部除外） | 10, 11, 12, 13, 14 | 制御文字・変換不能 | 0x0E（SO）、0x0F（SI）は除外 |
| `0x16`–`0x24` | 16, 17, ..., 24 | 制御文字・変換不能 | |
| `0x26`–`0x3F`（一部除外） | 26, 27, ..., 3F | 制御文字・変換不能 | 0x0E（SO）、0x0F（SI）は除外 |
| `0xFF` | FF | 変換不能 | |

**除外コード（不正検知対象外）**

| コード | 理由 |
|---|---|
| `0x0D` | CR: 行区切りコードとして処理 |
| `0x0E` | SO: SO/SI 検証で別途評価 |
| `0x0F` | SI: SO/SI 検証で別途評価 |
| `0x15` | NL: 行区切りコードとして処理 |
| `0x25` | NL25: 行区切りコードとして処理 |
| `0x40`–`0xFE` | 通常の EBCDIC 文字コード範囲 |

## 4. 出力ファイル1 — 変換後 UTF-8 ファイル

### 4-1. ファイルパス決定ルール

**単一ファイル指定の場合**

```
入力: -i /input/MYPROG  -o /output
出力: /output/MYPROG.txt  （拡張子なし → .txt 付与）
```

**ディレクトリ指定の場合**

```
入力: -i /input  -o /output
入力ファイル: /input/src/MYPROG
出力: /output/src/MYPROG.txt
```

### 4-2. 拡張子決定ルール

| 条件 | 出力拡張子 |
|---|---|
| `-e` オプション指定あり | 指定した拡張子（元の拡張子を置換） |
| `-e` 未指定 かつ 入力ファイルに拡張子あり | 元の拡張子をそのまま保持 |
| `-e` 未指定 かつ 入力ファイルに拡張子なし | `.txt` を付与 |

**例**

| 入力ファイル名 | `-e` オプション | 出力ファイル名 |
|---|---|---|
| `MYPROG` | なし | `MYPROG.txt` |
| `MYPROG` | `cbl` | `MYPROG.cbl` |
| `MYPROG.cbl` | なし | `MYPROG.cbl` |
| `MYPROG.cbl` | `txt` | `MYPROG.txt` |

### 4-3. ファイル内容仕様

| 項目 | 仕様 |
|---|---|
| 文字コード | UTF-8（BOM なし） |
| 改行コード | LF（`0x0A`） |
| 変換不能文字 | `?`（U+003F）に置換して出力 |
| 固定長モード行末 | 各レコードの末尾に LF を付与 |
| バイトストリームモード行末 | 行区切りコードを LF に変換 |
| 既存ファイル | 上書き |

```markdown
## 5. 不正文字検知詳細

| ファイル名 | 行番号 | オフセット | EBCDIC HEX値 | UTF-8 HEX値 | 変換後文字 | 検知理由 |
|-----------|-------|-----------|-------------|------------|----------|---------|
| C:\work\input\MYPROG2 | 5 | 10 | 0x1F | - | - | 制御文字 |
| C:\work\input\MYPROG2 | 12 | 0 | 0x0E | - | - | SO/SI不正 |
| C:\work\input\MYPROG2 | 20 | 34 | 0xFF | - | - | 変換不能 |
```

### 5-8. セクション6: SBCS コード変換対応表 — フォーマット例

```markdown
## 6. SBCS コード変換対応表（付録）

コードページ: IBM-930

| EBCDIC HEX値 | UTF-8 HEX値 | UTF-8文字 | 逆変換 EBCDIC HEX値 |
|-------------|------------|----------|-------------------|
| 0x00 | 0x00 | - | 0x00 |
| 0x01 | 0x01 | - | 0x01 |
| ...（256 エントリ）... |
| 0x41 | 0x20 |   | 0x41 |
| 0x42 | - | - | - |
```

**SBCS コード変換対応表の生成ルール**

1. `0x00` ～ `0xFF` の各バイト 1 バイトを、EBCDIC コードページで変換する（SBCS として扱う）。
2. SO（`0x0E`）以降は DBCS コードが続く可能性があるが、付録では SBCS 変換のみを対象とする。
3. `Charset.newDecoder()` に `CodingErrorAction.REPLACE` を設定して変換し、`?` に置換された場合は変換不能として `-` を記録する。
4. 逆変換は変換後の UTF-8 文字を再度 EBCDIC に変換して得る。変換不能の場合は `-` を記録する。
5. 変換後文字が制御文字や表示不能文字の場合は `-` を記録する。

## 6. 標準出力（進捗表示）

変換処理の進捗を以下のフォーマットで標準出力に表示する。

| タイミング | 出力内容 | フォーマット例 |
|---|---|---|
| 変換処理開始時 | 処理開始メッセージ | `[INFO] E2UConverter started.` |
| 各ファイル変換完了時 | ファイル名とステータス | `[INFO] OK      : C:\work\input\MYPROG` |
| 各ファイル WARNING 時 | ファイル名と不正文字件数 | `[WARN] WARNING : C:\work\input\MYPROG2 (3 invalid char(s))` |
| 各ファイル ERROR 時 | ファイル名とエラー内容 | `[ERROR] ERROR   : C:\work\input\MYPROG3 - Access denied` |
| 全処理完了時 | レポートパスと統計 | `[INFO] Report: C:\work\output\report.md` |
| 全処理完了時 | 完了メッセージ | `[INFO] Done. Total: 10, OK: 8, WARNING: 1, ERROR: 1` |

## 7. 標準エラー出力

致命的エラー発生時のメッセージフォーマット。

| エラー種別 | メッセージフォーマット例 |
|---|---|
| 必須オプション未指定 | `ERROR: -i option is required.` |
| 入力パス不存在 | `ERROR: Input path does not exist: C:\work\input` |
| コードページ不正 | `ERROR: Unsupported codepage: IBM-9999` |
