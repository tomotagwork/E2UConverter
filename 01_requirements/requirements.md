# 要件定義書 — EBCDIC → UTF-8 変換ツール (E2UConverter)

## 1. 概要・目的

z/OS 上で EBCDIC コードにより管理されているソースコード（PDS メンバー）を UTF-8 に変換し、Git による版管理に移行できるようにするための変換ツールを作成する。

| 項目 | 内容 |
|------|------|
| 成果物 | PC 上で動作する CLI ツール（Java 実装） |
| 主目的 | EBCDIC ファイル → UTF-8 ファイルへの一括変換 |
| 副目的 | 変換不能文字・制御文字の検知とレポート出力 |

---

## 2. 前提条件

- 変換対象の EBCDIC ファイルは、z/OS 上の PDS メンバーをバイナリーモードで PC に転送済みであること。
- 変換を実行する PC に Java 実行環境（JRE 11 以上を想定）が導入済みであること。
- EBCDIC コードページは日本語を含むもの（IBM-930, IBM-939, IBM-1390, IBM-1399 等）を想定する。
- Java の文字コードコンバーター名をそのままオプションで指定する。

---

## 3. アプリケーション実行形態

- アプリケーションは `java` コマンドで CLI から実行できるものとする。
- ビルドツールには **Maven** を使用し、依存ライブラリを含む **実行可能 JAR（fat JAR）** としてビルドする。
- プロジェクトルートに Windows 向けの **バッチファイル（.bat）** を用意し、簡便に実行できるようにする。

---

## 4. コマンドラインオプション一覧

| オプション | 必須/任意 | 説明 | デフォルト |
|-----------|----------|------|----------|
| `-i <path>` | 必須 | 入力ファイルまたはディレクトリのパス | — |
| `-o <dir>` | 必須 | 変換後ファイルの出力先ディレクトリ | — |
| `-c <codepage>` | 任意 | EBCDIC コードページ（Java コンバーター名） | `IBM-930` |
| `-m <mode>` | 任意 | 入力モード。`fixed`=固定長レコード / `stream`=バイトストリーム | `fixed` |
| `-l <length>` | 任意 | 固定長レコードモード時の LRECL（バイト数） | `80` |
| `-r` | 任意 | ディレクトリ入力時にサブディレクトリを再帰処理する | 無効（フラグ） |
| `-e <ext>` | 任意 | 出力ファイルの拡張子（ドットなし、例: `txt`） | 後述 |
| `-R <file>` | 任意 | レポートの出力先ファイルパス（相対または絶対） | `<出力先ディレクトリ>/report.md` |

### 出力拡張子の決定ルール

1. `-e` オプションが指定されている場合 → 入力ファイルの拡張子に関わらず、指定拡張子に置き換える。
2. `-e` オプションが未指定で、入力ファイルに拡張子がある場合 → 元の拡張子をそのまま保持する。
3. `-e` オプションが未指定で、入力ファイルに拡張子がない場合 → `.txt` を付与する。

### 必須オプション未指定時の動作

`-i` または `-o` が未指定の場合、エラーメッセージを標準エラー出力に表示して処理を中止する。

---

## 5. 入力ファイル仕様

### 5-1. 入力対象の指定

- `-i` に単一ファイルを指定した場合、そのファイルのみを処理する。
- `-i` にディレクトリを指定した場合、ディレクトリ直下のファイルを処理する。
  - `-r` オプションを指定した場合、サブディレクトリ配下のファイルも再帰的に処理する。

### 5-2. 入力モード

#### 固定長レコードモード（`-m fixed`、デフォルト）

- ファイルを `-l` で指定したバイト数（デフォルト 80 バイト）の固定長レコードとして読み込む。
- レコードの区切りは改行コードではなく、バイト数によって決定する。
- LRECL に満たないバイト数しかない最終レコードも 1 レコードとして処理する。

#### バイトストリームモード（`-m stream`）

- ファイル全体をバイトストリームとして読み込む。
- EBCDIC の `x'0D'`（CR）、`x'15'`（NL）、`x'25'`（NL25）を行区切りコードとして扱う。
- 行区切りコードが検出された場合、UTF-8 出力では LF（`x'0A'`）に変換する。

---

## 6. 不正文字検知仕様

### 6-1. 共通定義

| EBCDIC コード | 意味 |
|-------------|------|
| `x'0D'` | 改行コード（CR） |
| `x'15'` | 改行コード（NL） |
| `x'25'` | 改行コード（NL25） |
| `x'0E'` | Shift-Out（SO）: DBCS 開始 |
| `x'0F'` | Shift-In（SI）: DBCS 終了 |

- 指定コンバーターで文字として変換できなかったコードは、不正検知の対象とする。

### 6-2. 固定長レコードモードにおける不正文字検知

#### 制御文字・変換不能コードの検知

以下のコードは制御文字または文字として認識できないコードとして不正検知の対象とする。

| 範囲 | 備考 |
|------|------|
| `x'00'` ～ `x'0D'` | 制御文字（x'0D' を含む。固定長モードでは改行コードも不正） |
| `x'10'` ～ `x'3F'` | 制御文字・変換不能コード（x'0E'、x'0F' を除く） |
| `x'FF'` | 変換不能コード |

> `x'0E'`（SO）と `x'0F'`（SI）は DBCS シフトコードとして扱うため、上記検知範囲から除外し、後述の SO/SI 検証ルールで別途評価する。

#### コードページ未定義文字の検知

`x'40'` ～ `x'FE'` の範囲のバイトのうち、指定コードページで文字として変換できないバイト（変換結果が代替文字 `?` になるバイト）は、不正文字として検知し、検知理由を「未定義文字」としてレポートに記録する。出力ファイルへの書き込みは Java Charset の REPLACE 設定（`?` への置換）に任せる。

#### SO/SI ペアの検証（1 レコード単位）

以下のいずれかに該当する場合、そのコードを不正検知の対象とする。

1. SO と SI のペアが 1 レコード内で正しい順序（SO → … → SI）で現れていない場合。
2. SO/SI のペアがレコードをまたがっている場合（レコード末尾で SI が閉じていない場合）。
3. SO の直後に SI が現れる場合（DBCS の中身が空の場合）。

### 6-3. バイトストリームモードにおける不正文字検知

#### 制御文字・変換不能コードの検知

以下のコードは制御文字または文字として認識できないコードとして不正検知の対象とする。

| 範囲 | 備考 |
|------|------|
| `x'00'` ～ `x'0C'` | 制御文字 |
| `x'10'` ～ `x'14'` | 制御文字・変換不能コード（x'0E'、x'0F' を除く） |
| `x'16'` ～ `x'24'` | 制御文字・変換不能コード |
| `x'26'` ～ `x'3F'` | 制御文字・変換不能コード（x'0E'、x'0F' を除く） |
| `x'FF'` | 変換不能コード |

> `x'0D'`、`x'15'`、`x'25'` は行区切りコードとして処理するため除外する。
> `x'0E'`（SO）と `x'0F'`（SI）は DBCS シフトコードとして扱うため除外し、後述の SO/SI 検証ルールで別途評価する。

#### コードページ未定義文字の検知

`x'40'` ～ `x'FE'` の範囲のバイトのうち、指定コードページで文字として変換できないバイト（変換結果が代替文字 `?` になるバイト）は、不正文字として検知し、検知理由を「未定義文字」としてレポートに記録する。出力ファイルへの書き込みは Java Charset の REPLACE 設定（`?` への置換）に任せる。

#### SO/SI ペアの検証（1 行単位）

以下のいずれかに該当する場合、そのコードを不正検知の対象とする。

1. SO と SI のペアが 1 行内で正しい順序（SO → … → SI）で現れていない場合。
2. SO/SI のペアが行をまたがっている場合（行末尾で SI が閉じていない場合）。
3. SO の直後に SI が現れる場合（DBCS の中身が空の場合）。

## 7. 出力ファイル1 — 変換後 UTF-8 ファイル

### 7-1. ファイルパスの決定

- 出力先ディレクトリ（`-o`）配下に、入力ファイルの相対パス構造を再現した場所にファイルを出力する。
  - 例: 入力が `/input/src/MYPROG`、`-i /input` の場合 → 出力は `<-o>/src/MYPROG.txt`
- ファイル名（拡張子の決定）は「4. コマンドラインオプション一覧」の「出力拡張子の決定ルール」に従う。

### 7-2. ファイル内容

- 入力ファイルの各バイトを指定コードページで変換した UTF-8 文字列を出力する。
- SO/SI を含む DBCS コードは Java コンバーターにより Unicode 文字に変換される。変換結果をそのまま出力する（SO/SI に相当するコードは UTF-8 には存在しないため、変換結果から自動的に除去される）。
- 変換できなかった文字は代替文字 `?` に置換して出力する。
- 変換不能文字が含まれる場合でも変換ファイルは必ず出力する。
- 固定長レコードモードの場合、各レコードの末尾に LF（`\n`）を付与して行として出力する。
- バイトストリームモードの場合、行区切りコード（`x'0D'`/`x'15'`/`x'25'`）を LF（`\n`）に変換して出力する。
- 同名の出力ファイルが既に存在する場合は上書きする。

---

## 8. 出力ファイル2 — Markdown レポート

### 8-1. レポートファイルの出力先

- `-R` オプションで相対パスまたは絶対パスを指定できる。
- `-R` が未指定の場合、`-o` で指定した出力先ディレクトリに `report.md` として出力する。

### 8-2. レポートの構成

#### セクション1: 実行サマリー

以下の情報を含む。

| 項目 | 内容 |
|------|------|
| 処理日時 | 変換処理を実行した日時 |
| 処理対象 | `-i` で指定した入力ファイル/ディレクトリのパス |
| 出力先 | `-o` で指定した出力先ディレクトリ |
| 使用コードページ | `-c` で指定したコードページ名 |
| 入力モード | `fixed`（LRECL=xx）または `stream` |
| 再帰処理 | 有効/無効 |

#### セクション2: 処理統計

| 項目 | 内容 |
|------|------|
| 処理ファイル総数 | 変換処理を試みたファイルの総数 |
| 変換成功ファイル数 | 不正文字なしで変換できたファイルの数 |
| 不正文字検知ファイル数 | 1 件以上の不正文字を含んでいたファイルの数 |
| 不正文字検知総件数 | 全ファイルにおける不正文字の総検知件数 |
| エラーファイル数 | 処理に失敗したファイルの数 |

#### セクション3: 処理ファイル一覧

処理したファイルの一覧を表形式で出力する。

| 列 | 内容 |
|----|------|
| ファイル名 | 入力ファイルのパス |
| ステータス | `OK` / `WARNING`（不正文字あり）/ `ERROR` |
| 不正文字検知件数 | そのファイルで検知した不正文字の件数 |

#### セクション4: エラーファイル一覧

処理に失敗したファイルとその理由を表形式で出力する。

| 列 | 内容 |
|----|------|
| ファイル名 | 入力ファイルのパス |
| エラー内容 | エラーの理由（例: ファイル読み込みエラー、コードページ不正 等） |

#### セクション5: 不正文字検知詳細

全ファイルの不正文字検知情報を 1 つの表形式で出力する。

| 列 | 内容 |
|----|------|
| ファイル名 | 入力ファイルのパス |
| 行番号（レコード番号） | 不正文字が検知された行番号（1 始まり） |
| オフセット | 行頭（レコード先頭）からのバイトオフセット（0 始まり） |
| EBCDIC HEX 値 | 変換前の EBCDIC コード（HEX 表記、例: `0x1F`） |
| UTF-8 HEX 値 | 変換後の UTF-8 コード（HEX 表記。変換不能の場合は `-`） |
| 変換後文字 | 変換後の文字（表示可能な場合のみ。不能の場合は `-`） |
| 検知理由 | 検知理由（例: 制御文字、SO/SI 不正、変換不能 等） |

#### セクション6: SBCS コード変換対応表（付録）

EBCDIC の SBCS コード `x'00'` ～ `x'FF'` の各コードについて、以下の情報を表形式で出力する（256 エントリ）。

| 列 | 内容 |
|----|------|
| EBCDIC HEX 値 | EBCDIC コードの HEX 表記（例: `0x41`） |
| UTF-8 HEX 値 | コンバーターによる変換後の UTF-8 コード（HEX 表記） |
| UTF-8 文字 | 変換後の文字（表示可能な場合のみ） |
| 逆変換 EBCDIC HEX 値 | UTF-8 を再度 EBCDIC に変換した結果（HEX 表記。変換不能の場合は `-`） |

## 9. エラー処理仕様

| エラー状況 | 動作 |
|-----------|------|
| `-i` または `-o` オプション未指定 | エラーメッセージを標準エラー出力に表示し、処理中止（exit code: 1） |
| 指定された入力ファイル/ディレクトリが存在しない | エラーメッセージを表示し、処理中止（exit code: 1） |
| 指定されたコードページが Java で認識されない | エラーメッセージを表示し、処理中止（exit code: 1） |
| 個別ファイルの読み込みエラー | そのファイルをエラーとしてレポートに記録し、次のファイルの処理を継続する |
| 出力先ディレクトリが存在しない | 自動的に作成して処理を継続する |
| 変換不能文字を検知した場合 | `?` に置換して変換処理を継続し、レポートに記録する |
| コードページ上の未定義文字を検知した場合 | Java Charset の REPLACE 設定により `?` に置換して変換処理を継続し、レポートに「未定義文字」として記録する |

---

## 10. 非機能要件 / 制約

| 項目 | 内容 |
|------|------|
| 実装言語 | Java（JDK 11 以上を想定） |
| ビルドツール | Maven |
| 成果物 | 実行可能 JAR（fat JAR） |
| 実行スクリプト | Windows バッチファイル（.bat） |
| 文字コードコンバーター | Java 標準の `java.nio.charset.Charset` を使用（IBM-930 等） |
| 対象プラットフォーム | Windows PC |
| レポート形式 | Markdown (.md) |
| ログ出力 | 処理の進捗は標準出力に表示する（詳細レベルは今後検討） |
