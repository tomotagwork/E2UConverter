## 3. 不具合一覧（修正済）

### BUG-001: 変換不能文字（0xFF）が `?` に置換されず U+009F が出力される ✅修正済

| 項目 | 内容 |
|------|------|
| 不具合ID | BUG-001 |
| 影響テストケース | TG-5-001, TG-9-001 |
| 重要度 | 中 |
| 再現条件 | IBM-930 コードページを使用し、入力に `0xFF` バイトを含む場合 |
| 期待動作 | 出力ファイルの該当位置が `?`（U+003F）に置換される |
| 実際の動作（修正前） | U+009F（EBCDIC `0xFF` → UTF-8 `0xC2 0x9F`、制御文字）が出力される |
| 修正内容 | [`FileConverter.replaceInvalidSoSiBytes()`](03_implementation/e2uconverter/src/main/java/com/example/e2uconverter/converter/FileConverter.java:183) を拡張し、「変換不能」として検知したバイトオフセットもコードページの `?` に相当する EBCDIC バイト（IBM-930: `0x6F`）に置換するよう修正。また [`getQuestionByte()`](03_implementation/e2uconverter/src/main/java/com/example/e2uconverter/converter/FileConverter.java:213) を追加し、コードページ固有の `?` バイトを動的に取得するよう実装。 |
| ステータス | **修正済・再テスト合格** |

**原因分析:**

IBM-930 の Java Charset は `0xFF` を U+009F（MEDIUM MESSAGE WAITING、制御文字）にマッピングする。U+009F は有効な Unicode コードポイントであるため、`CharsetDecoder` の `onUnmappableCharacter(CodingErrorAction.REPLACE)` がトリガーされず、`?` に置換されない。

旧実装では `replaceInvalidSoSiBytes()` が SO/SI 不正のみを EBCDIC `0x3F`（→ U+001A に変換されてしまう）に置換しており、変換不能バイトは未対処だった。

## 5. 不合格テストケースの詳細（再テスト要）

### TG-5-001 / TG-9-001: 変換不能文字の `?` 置換問題（BUG-001）

**現象:**
- IBM-930 コードページで `0xFF` バイトを含む入力を変換すると、出力ファイルの該当位置に `?`（U+003F）ではなく U+009F（UTF-8: `0xC2 0x9F`）が出力される。
- テスト計画の期待値: `出力ファイルの該当位置が ? に置換`

**根本原因:**
IBM-930 の Java Charset（`IBM-930`）が `0xFF` を U+009F にマッピングするため、`CharsetDecoder.onUnmappableCharacter(REPLACE)` が動作しない。U+009F は制御文字だが「マッピング不能」ではないため REPLACE が適用されない。

**修正対象コード:**
[`FileConverter.java`](03_implementation/e2uconverter/src/main/java/com/example/e2uconverter/converter/FileConverter.java:186) の `replaceInvalidSoSiBytes()` メソッドを拡張し、「変換不能」として検知したバイトオフセットも `0x3F` に置換するよう修正が必要。

**修正内容（暫定案）:**

```java
private byte[] replaceInvalidSoSiBytes(byte[] record, List<InvalidCharEntry> entries, int recordNo) {
    java.util.Set<Integer> invalidOffsets = new java.util.HashSet<>();
    for (InvalidCharEntry e : entries) {
        if (e.getRecordNo() == recordNo) {
            // SO/SI不正 および 変換不能 の両方を置換対象とする
            if ("SO/SI不正".equals(e.getReason()) || "変換不能".equals(e.getReason())) {
                invalidOffsets.add(e.getOffset());
            }
        }
    }
    if (invalidOffsets.isEmpty()) return record;
    byte[] copy = record.clone();
    for (int off : invalidOffsets) {
        if (off < copy.length) copy[off] = (byte) 0x3F;
    }
    return copy;
}
```
